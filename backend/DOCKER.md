# Docker инструкция

## Сборка и запуск

### Быстрый старт

```bash
# Сборка образа
docker compose build

# Запуск контейнера
docker compose up -d

# Просмотр логов
docker compose logs -f backend

# Остановка контейнера
docker compose down
```

## Структура

- **Dockerfile** - определение образа приложения
- **docker-compose.yml** - конфигурация для запуска с монтированием .env
- **.dockerignore** - файлы, исключаемые из образа

## Монтирование .env

Файл `.env` автоматически монтируется в контейнер как read-only volume:
- Путь в контейнере: `/app/.env`
- Путь на хосте: `./.env`

## Volumes

Следующие директории монтируются как volumes:
- `./.env` → `/app/.env` (read-only)
- `./uploads` → `/app/uploads` (для загруженных файлов)
- `./error.log` → `/app/error.log` (для логов)

## Переменные окружения

Приложение использует переменные из `.env` файла. Убедитесь, что файл `.env` существует и содержит все необходимые настройки (см. `.env.example`).

## Порты

По умолчанию приложение запускается на порту 8000. Можно изменить через переменную `SERVER_PORT` в `.env`.

## Требования

- Docker и Docker Compose должны быть установлены
- Файл `.env` должен существовать в корне проекта
- База данных PostgreSQL должна быть доступна (настроена в `.env`)
